## Creacion Agente de Ventas

En la firma financiera FinPulse, los ejecutivos comerciales pierden el 80% de su jornada en tareas administrativas: actualizando etapas de oportunidades manualmente, buscando historiales de interacciones dispersos y tratando de identificar qué clientes contactar cada día. A pesar de tener un catálogo de productos y manuales de forecast, la falta de una guía proactiva genera que las oportunidades de inversión se estanquen. El desafío es construir un agente de ventas: un agente interno que automatice el seguimiento y la gestión del pipeline comercial. 


#### Objetivo
habilidades como “arquitecto de IA”, integrando de manera estratégica distintas capacidades de la plataforma para 
construir un agente inteligente, útil y confiable. Considera lo siguiente: 
1. Topics & agent builder: Configuración de componentes para diferenciar si la solicitud es una consulta de historial, un cambio de etapa o un pronóstico de ventas. 
2. Data cloud & search: Uso de connections con los objetos de Actividad y Oportunidad para proporcionar insights basados en datos reales.  
3. Flow builder: Creación de flujos lógicos para la actualización rápida de registros y generación de reportes automáticos de forecast.

#### Desarrollo
El proceso de EDA se desarrollará en varios pasos detallados a continuación:

1. Desarrolla un vídeo demo de 3 minutos máximo, en él muestra al agente resolviendo 
con éxito por lo menos 3 escenarios diferentes, (puedes entre los siguientes o incluir 
algún otro nuevo): 
**scenario 1 (seguimiento)**: El usuario pregunta: "¿Qué oportunidades debo 
contactar hoy?"; el agente muestra la lista con actividades próximas. 
**Escenario 2 (actualización)**: El usuario pide: "Cambia la etapa de oportunidad 
X a Negociación"; el agente ejecuta y registra el evento. 
**Escenario 3 (forecast)**: El usuario consulta: "¿Cuál es mi forecast trimestral?"; 
el agente genera el reporte probable de cierre.
